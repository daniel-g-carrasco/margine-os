# Stato del 2026-03-30

## Fix da riportare dal sistema attuale

- `Walker` resta la scelta di default per il launcher.
- `elephant` non va lanciato con `exec-once`: sul sistema attuale si è stabilizzato usando un servizio utente `systemd` con `Restart=always`.
- Il menu VPN in `waybar` deve usare `nmcli --ask` per i profili `OpenVPN`, così le credenziali vengono richieste in TTY invece di fallire in silenzio.
- Per `WireGuard`, il menu deve mostrare un errore chiaro quando il kernel in esecuzione non ha più i moduli installati, invece di limitarsi a fallire.
- La catena `hyprlock` + fingerprint su Framework 13 richiede una mitigazione esplicita sul resume: il candidato attuale è il restart di `fprintd` via hook `systemd-sleep`.
- Il login path target è stato chiuso come `greetd + tuigreet`, con autologin iniziale verso Hyprland seguito da `hyprlock`.
- La connettività è stata estratta da `hyprland-core` e resa un layer di sistema autonomo, con `NetworkManager + iwd + Bluetooth + VPN`, più provisioning del regulatory domain.
- Il desktop layer `Hyprland` è stato versionato come payload utente separato, invece di restare implicito nella macchina corrente.

## Nota operativa

Questi punti sono stati isolati sul sistema reale. La loro traduzione in
`Margine` e' iniziata con:

- layer `connectivity-stack`;
- ADR sul login path `greetd + tuigreet + autologin iniziale`;
- provisioner per `greetd`;
- provisioner per i runtime tool utente;
- hook versionato per il resume di `fprintd`;
- provisioner di sistema per `NetworkManager + iwd + regdom`;
- provisioner del desktop layer `Hyprland`.
