# Status as of 2026-03-30

## Fix to be reported from current system

- `Walker` remains the default choice for the launcher.
- `elephant` should not be launched with `exec-once`: on the current system it has stabilized using a `systemd` user service with `Restart=always`.
- The VPN menu in `waybar` should use `nmcli --ask` for `OpenVPN` profiles, so credentials are requested in TTY instead of failing silently.
- For `WireGuard`, the menu should show a clear error when the running kernel no longer has modules installed, instead of just failing.
- The `hyprlock` + fingerprint chain on Framework 13 requires explicit mitigation on the resume: the current candidate is restarting `fprintd` via hook `systemd-sleep`.
- The login path target was closed as `greetd + tuigreet`, with initial autologin to Hyprland followed by `hyprlock`.
- Connectivity has been extracted from `hyprland-core` and made a self-contained system layer, with `NetworkManager + iwd + Bluetooth + VPN`, plus regulatory domain provisioning.
- The `Hyprland` desktop layer has been versioned as a separate user payload,
  instead of remaining implicit on the current machine.

## Operational notes

These points have been isolated on the real system. Their translation into
`Margine` started with:

- layer `connectivity-stack`;
- ADR on login path `greetd + tuigreet + initial autologin`;
- provisioner for `greetd`;
- provisioner for user runtime tools;
- versioned hook for `fprintd` resume;
- system provisioner for `NetworkManager + iwd + regdom`;
- `Hyprland` desktop layer provisioner.
- selective migration allowlist for `home` and `system` configurations;
- application layer versioned for `Neovim`, `Kitty`, `mimeapps`, `user-dirs`
  and global `update-all`;
- baseline `Firefox` via system policy;
- system baseline fingerprint (`/etc/pam.d/hyprlock`, `/etc/fprintd.conf`)
  versioned;
- generation of `Limine` entry recoveries starting from `Snapper` snapshots.
- initial application review closed for `Firefox`, `Thunderbird`, `Darktable`,
  `VS Code` and `Timeshift/Snapper`, with `Darktable` migrated selectively.
- conservative baseline of `Timeshift` versioned via `default.json`,
  without UUID and without automatic snapshots.
- `update-all` now also creates an initial automatic snapshot for the entire
  update flow, in addition to `snap-pac` for `pacman`.
- photography/color management baseline opened and translated into versioned layer:
  `darktable + argyllcms + displaycal + colord`, plus ICC assets selected.
- explicit choice on color management in `Hyprland`: `Margine v1` remains
  `colord + app-first`, with ICC composer-level optional and not active
  by default.
- the `browser + mail defaults` block is now closed: `Firefox` browser
  baseline,
  `Thunderbird` official as baseline mailer, `mimeapps.list` aligned to
  real-system desktop IDs.
- "selective migration of personal profiles" theme closed with matrix
  app-by-app: what to version, what to exclude and what to postpone for future review.
- baseline workstation updated: `kitty` remains the only baseline terminal,
  `Ghostty` leaves the `v1` perimeter, and a layer dedicated to tooling is created from
  coding and administration (`tmux`, `opencode`, system monitor, utilities
  CLI).
- baseline remote access and closed defense network: `openssh` present,
  `ufw` chosen as firewall `v1`, policy `deny incoming / allow outgoing`,
  `sshd` ready but not automatically exposed.
- closed printing/scanning baseline: `CUPS + Avahi + ipp-usb` print side,
  `SANE + sane-airscan` scanner side, with `system-config-printer` and
  `simple-scan` as management tools.
- positive real-world validation of the printing/scanning baseline
  `Brother MFC-L2860DWE`: print via `IPP`, scan via `eSCL`, then
  the `driverless-first` path is confirmed by a concrete use case.
- extended runtime audit in progress/closed for the riskiest blocks:
power, resume, audio, network, Bluetooth, screenshot/recording, boot/recovery,
  virtualization and containers.
- a real gap emerged compared to the `Margine` vision: the current machine has
  Secure Boot and UKI active, but `Snapper` is not really configured and the
  virtualization does not yet have a complete baseline.
- an installation gap also emerged in the project: the initial bootstrap still
  did not close the `Limine + UKI` boot chain, so it was not yet possible to
  treat a `live ISO -> first boot` test as realistic.
- added the `QEMU + OVMF + Arch ISO` harness as the initial installer
  validation path, before any testing on the real laptop.
