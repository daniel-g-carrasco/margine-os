# Stato del 2026-03-30

## Fix da riportare dal sistema attuale

- `Walker` resta la scelta di default per il launcher.
- `elephant` non va lanciato con `exec-once`: sul sistema attuale si è stabilizzato usando un servizio utente `systemd` con `Restart=always`.
- Il menu VPN in `waybar` deve usare `nmcli --ask` per i profili `OpenVPN`, così le credenziali vengono richieste in TTY invece di fallire in silenzio.
- Per `WireGuard`, il menu deve mostrare un errore chiaro quando il kernel in esecuzione non ha più i moduli installati, invece di limitarsi a fallire.
- La catena `hyprlock` + fingerprint su Framework 13 richiede una mitigazione esplicita sul resume: il candidato attuale è il restart di `fprintd` via hook `systemd-sleep`.
- Il login path target è stato chiuso come `greetd + tuigreet`, con autologin iniziale verso Hyprland seguito da `hyprlock`.
- La connettività è stata estratta da `hyprland-core` e resa un layer di sistema autonomo, con `NetworkManager + iwd + Bluetooth + VPN`, più provisioning del regulatory domain.
- Il desktop layer `Hyprland` è stato versionato come payload utente separato, invece di restare implicito nella macchina corrente.

## Nota operativa

Questi punti sono stati isolati sul sistema reale. La loro traduzione in
`Margine` e' iniziata con:

- layer `connectivity-stack`;
- ADR sul login path `greetd + tuigreet + autologin iniziale`;
- provisioner per `greetd`;
- provisioner per i runtime tool utente;
- hook versionato per il resume di `fprintd`;
- provisioner di sistema per `NetworkManager + iwd + regdom`;
- provisioner del desktop layer `Hyprland`.
- allowlist di migrazione selettiva per configurazioni `home` e `system`;
- layer applicativo versionato per `Neovim`, `Kitty`, `mimeapps`, `user-dirs`
  e `update-all` globale;
- baseline `Firefox` via policy di sistema;
- baseline fingerprint di sistema (`/etc/pam.d/hyprlock`, `/etc/fprintd.conf`)
  versionata;
- generazione delle entry recovery `Limine` a partire dagli snapshot `Snapper`.
- review applicativa iniziale chiusa per `Firefox`, `Thunderbird`, `Darktable`,
  `VS Code` e `Timeshift/Snapper`, con `Darktable` migrato in forma selettiva.
- baseline conservativa di `Timeshift` versionata tramite `default.json`,
  senza UUID e senza snapshot automatici.
- `update-all` ora prevede anche uno snapshot automatico iniziale prima
  dell'intero flusso di aggiornamento, oltre a `snap-pac` per `pacman`.
- baseline fotografia/color management aperta e tradotta in layer versionato:
  `darktable + argyllcms + displaycal + colord`, piu' ICC asset selezionati.
- scelta esplicita sul color management in `Hyprland`: `Margine v1` resta
  `colord + app-first`, con ICC compositor-level opzionale e non attivo
  di default.
- blocco `browser + mail defaults` chiuso: `Firefox` browser baseline,
  `Thunderbird` ufficiale come mailer baseline, `mimeapps.list` allineato ai
  desktop IDs reali del sistema.
- tema "migrazione selettiva dei profili personali" chiuso con matrice
  app-per-app: cosa versionare, cosa escludere e cosa rinviare a review futura.
- baseline workstation aggiornata: `kitty` resta il solo terminale baseline,
  `Ghostty` esce dal perimetro `v1`, e nasce un layer dedicato al tooling da
  coding e amministrazione (`tmux`, `opencode`, monitor di sistema, utility
  CLI).
- baseline accesso remoto e rete difensiva chiusa: `openssh` presente,
  `ufw` scelto come firewall `v1`, policy `deny incoming / allow outgoing`,
  `sshd` pronto ma non esposto automaticamente.
- baseline stampa/scanner chiusa: `CUPS + Avahi + ipp-usb` lato stampa,
  `SANE + sane-airscan` lato scanner, con `system-config-printer` e
  `simple-scan` come strumenti di gestione.
- validazione reale positiva della baseline stampa/scanner su
  `Brother MFC-L2860DWE`: stampa via `IPP`, scanner via `eSCL`, quindi il
  percorso `driverless-first` e' confermato da un caso d'uso concreto.
- audit runtime esteso in corso/chiuso per i blocchi piu' rischiosi:
  power, resume, audio, rete, Bluetooth, screenshot/recording, boot/recovery,
  virtualizzazione e container.
- emerso un gap reale rispetto alla visione `Margine`: la macchina attuale ha
  Secure Boot e UKI attivi, ma `Snapper` non e' davvero configurato e la
  virtualizzazione non ha ancora una baseline completa.
- emerso anche un gap installativo nel progetto: il bootstrap iniziale non
  chiudeva ancora la boot chain `Limine + UKI`, quindi non era possibile
  considerare realistico un test `live ISO -> primo boot`.
- aggiunto l'harness `QEMU + OVMF + Arch ISO` come percorso di validazione
  iniziale dell'installer, prima di qualunque prova sul laptop reale.
