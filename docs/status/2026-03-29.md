# Status as of 2026-03-29

## Decisions made today

- human project name: `Margine`
- repository name: `margine-os`
- repo-first approach, not iso-first
- mandatory documentation
- mandatory teaching approach
- system replication via allowlist, not via full dump
- `Floorp` excluded
- `Firefox` included as main browser
- AUR allowed only as a reasoned exception
- `Koofr` is an AUR candidate to be explicitly evaluated
- ADR 0002 accepted:
  - `Limine`
  - `UKI`
  - `LUKS2`
  - `TPM2`
  - `Btrfs`
  - `Snapper`
  - `Secure Boot` as an explicit target
  - bootable snapshots as an explicit target
  - fallback to `systemd-boot` if validation fails
- ADR 0003 accepted:
  - `ESP` large mounted on `/boot`
  - rest of the disk in `LUKS2`
  - root `Btrfs`
  - dedicated subvolumes for recovery, VM and containers
  - `zram` as swap of `v1`
  - no hibernation in `v1`
  - `compress=zstd:3` as base
  - `fstrim.timer` instead of `discard`
  - explicit distinction between `rootful` and `rootless` workloads
  - initial manifest of target subvolumes already set
- ADR 0004 accepted:
  - validation by gate of the `Limine + UKI + Secure Boot + TPM2 + Snapper` chain
  - `mkinitcpio` as a starting point to generate `UKI`
  - `sbctl` as key and signature manager
  - initial policy `TPM2` on `PCR 7+11`
  - mandatory recovery key before enrollment `TPM2`
- ADR 0005 accepted:
  - separation between path `prod` and path `recovery`
  - `UKI` normal with embedded command line
  - `UKI` recovery without built-in command line
  - `Limine` to `EFI/BOOT`, `UKI` to `EFI/Linux`
  - `limine enroll-config` as part of the trust chain
- ADR 0006 accepted:
  - `Margine` as a playable layer over Arch rolling
  - no frozen fork of Arch
  - ordinary system update via normal Arch flow
  - update the repo only when the architecture or automation changes
- ADR 0007 accepted:
  - Final `limine.conf` as generated artifact
  - versioned template + machine facts + recovery block
  - final file on `ESP` not considered authoritative source
- ADR 0008 accepted:
  - `snapper + snap-pac` as baseline snapshot
  - automatic snapshots only for `root`
  - root timeline disabled in `v1`
  - `update-all` as canonical path over the safety net of `snap-pac`
  - explicit awareness that the `ESP` remains outside of the Btrfs snapshots
- ADR 0009 accepted:
  - `update-all` as canonical orchestrator
  - explicit order of update phases
  - distinction between hard and soft errors
  - first versioned script by `update-all`
- ADR 0010 accepted:
  - deploy on `ESP` only from already generated artifacts
  - preventive backup of overwritten files
  - no destructive cleaning of `ESP` in `v1`
- ADR 0011 accepted:
  - `limine enroll-config` executed after deployment on `ESP`
  - signature `sbctl` executed after `enroll-config`
  - `BOOTX64.EFI` and `UKI` signed as final artifacts on `ESP`
  - `sbctl verify` as final refresh trust check
- ADR 0012 accepted:
  - Secure Boot bootstrap separated by `update-all`
  - `sbctl enroll-keys -m` as conservative default
  - `sbctl` keys of type `file` in `v1`
  - `Setup Mode` required as an explicit prerequisite
- first `limine.conf` template added:
  - file versioned in `files/esp/EFI/BOOT/limine.conf.template`
  - static section in Git
  - recovery section foreseen as generated part
