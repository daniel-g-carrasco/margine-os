# Project Status - 2026-04-01

## Theme of the day

Close the gaps that emerged from the first real installation in the VM.

## Decisions

- `walker` + `elephant` enter the automatic AUR baseline.
- `fuzzel` becomes the official launcher fallback and shared picker for
  screenshot / recording menus.
- `update-all` tries to update the baseline AUR even without `yay`/`paru`,
using the project's internal AUR builder.
- `snapper` is actually provisioned: config `root`, `snap-pac`, snapshot
initial bootstrap and `snapper-cleanup.timer`.
- `libvirt` remains installed in the baseline, but `libvirtd` becomes opt-in:
it is only turned on via helper that correctly initializes the key
secret required by current libvirt releases.
- screenshot and recording definitively move to the official stack
`grim + slurp + satty + wf-recorder`, without depending on `hyprcap`.
- `elephant.service` is no longer enabled as a persistent user unit:
starts on-demand from the launcher after importing the Hyprland environment, avoiding
the regression where `Walker` stopped launching apps after reboot.

## Note

- Truly optional AUR exceptions remain separate: `davinci-resolve`,
  `koofr-desktop-bin`.
- On the real host system it is better to use the `margine-launcher` and wrappers
`margine-launcher-service`, so you actually test the fallback without it
  cambiare le bind.
- The first VM had also highlighted a useless warning of `greetd` on
`runfile`; the template now uses the default of upstream.
