# Stato progetto - 2026-04-01

## Tema del giorno

Chiudere i gap emersi dalla prima installazione reale in VM.

## Decisioni

- `walker` + `elephant` entrano nel baseline AUR automatico.
- `hyprlauncher` resta il fallback ufficiale, con config versionata e icone
  desktop esplicitamente abilitate.
- `update-all` prova ad aggiornare il baseline AUR anche senza `yay`/`paru`,
  usando il builder AUR interno del progetto.
- `snapper` viene provisionato davvero: config `root`, `snap-pac`, snapshot
  bootstrap iniziale e `snapper-cleanup.timer`.
- `libvirt` resta installato nella baseline, ma `libvirtd` diventa opt-in:
  viene acceso solo tramite helper che inizializza correttamente la chiave
  segreta richiesta dalle release attuali di libvirt.
- screenshot e recording passano definitivamente allo stack ufficiale
  `grim + slurp + satty + wf-recorder`, senza dipendere da `hyprcap`.
- `elephant.service` non viene piu' abilitato come unit utente persistente:
  parte on-demand dal launcher dopo l'import dell'ambiente Hyprland, evitando
  la regressione in cui `Walker` smetteva di lanciare app dopo il reboot.

## Note

- Le eccezioni AUR davvero opzionali restano separate: `davinci-resolve`,
  `koofr-desktop-bin`.
- Sul sistema host reale conviene usare i wrapper `margine-launcher` e
  `margine-launcher-service`, cosi' si testa davvero il fallback senza
  cambiare le bind.
- La prima VM aveva messo in evidenza anche un warning inutile di `greetd` sul
  `runfile`; il template ora usa il default di upstream.
