# Stato progetto - 2026-04-01

## Tema del giorno

Chiudere il gap tra launcher dichiarato e launcher realmente installato.

## Decisioni

- `walker` + `elephant` entrano nel baseline AUR automatico.
- `hyprcap` entra nello stesso baseline AUR per non rompere il workflow
  screenshot dichiarato dal desktop target.
- `hyprlauncher` resta il fallback ufficiale, con config versionata e icone
  desktop esplicitamente abilitate.
- `update-all` prova ad aggiornare il baseline AUR anche senza `yay`/`paru`,
  usando il builder AUR interno del progetto.

## Note

- Le eccezioni AUR davvero opzionali restano separate: `davinci-resolve`,
  `koofr-desktop-bin`.
- Sul sistema host reale conviene usare i wrapper `margine-launcher` e
  `margine-launcher-service`, cosi' si testa davvero il fallback senza
  cambiare le bind.
