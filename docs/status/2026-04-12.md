# Stato progetto - 2026-04-12

## Tema del giorno

Chiudere in modo definitivo il refresh della trust chain `Limine + sbctl` sul
path installato reale.

## Decisioni

- `update-all` usa ora file temporanei scrivibili per il rendering runtime di
  `limine.conf` e delle entry recovery, senza appoggiarsi a
  `/usr/local/lib/margine/build`.
- `update-all` reinstalla il binario `Limine` unsigned sul path EFI attivo
  prima di `limine enroll-config`.
- `update-all` rifirma il loader attivo subito dopo `enroll-config`.
- `refresh-efi-trust` adotta la stessa sequenza, cosi' il path manuale e il
  path di manutenzione ordinaria restano coerenti.

## Stato operativo verificato

Sul sistema host reale e' stato verificato con esito pulito:

- snapshot pre-update creato da `update-all`
- rigenerazione `UKI` completata
- `Config file BLAKE2B successfully enrolled`
- firma del loader attivo `/boot/EFI/Margine/BOOTX64.EFI`
- `sbctl verify` finale pulito

## Regola pratica stabilita

La sequenza da considerare invariabile e':

`deploy -> reinstall unsigned loader -> enroll-config -> sign -> verify`

Questa regola vale sia per:

- `refresh-efi-trust`
- `update-all`
- ogni altro call path che tocchi la trust chain EFI attiva
