# Margine Cross-Repo Audit - 2026-04-23

Repositories:

- public: `/home/daniel/dev/margine-os`
- personal: `/home/daniel/dev/margine-os-personal`

## Overall Finding

The two repositories are largely aligned in shared desktop code, theme code,
launcher code, and Walker packaging. The main problem is not random divergence;
the main problem is that both repositories contain the same large uncommitted
change set and both fail the same integrated static check.

## Shared Blockers

1. Both repos are dirty.

   Release work should pause until the current changes are reviewed and
   committed in coherent chunks.

2. Both repos fail `check-shell-and-manifests`.

   Shared failure:

   - `margine-launcher-rofi` `SC2002`
   - `margine-launcher-rofi` `SC2015`

3. Both repos carry generated bytecode in the source payload.

   Shared untracked files:

   - `files/home/.local/bin/__pycache__/gaming-split-lock-statuscpython-314.pyc`
   - `files/home/.local/bin/__pycache__/keep-awake-daemoncpython-314.pyc`
   - `files/home/.local/bin/__pycache__/notification-statuscpython-314.pyc`

4. Both repos carry the icon-theme migration in a transitional state.

   Shared pattern:

   - tracked deletion of `Margine-MoreWaita`
   - untracked `Margine-Adwaita`
   - host Rofi generated theme still points to `Margine-MoreWaita`

5. Both repos expose the same AUR builder sudo model.

   `aurbuilder` gets NOPASSWD root `pacman`. This should be treated as a
   security design decision, not an incidental implementation detail.

6. Both repos still contain Italian user-facing strings.

   This conflicts with the current English-only baseline requirement.

## Host Cross-Check

The host currently boots the public Arch product, not the personal CachyOS
product.

Current host observations:

- Secure Boot enabled
- Limine first in EFI boot order
- systemd-boot retained as fallback
- no failed system units
- no failed user units
- power key ignored, long press poweroff
- runtime baseline validator currently returns FAIL without precise failed
  subcheck
- host theme sync drift exists for Rofi generated icon theme
- journal shows a `hyprlock` coredump
- journal shows systemd random-seed permission warnings
