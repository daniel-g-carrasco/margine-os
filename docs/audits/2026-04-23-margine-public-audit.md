## Scope

This is a non-destructive audit. I inspected repository state, static syntax,
desktop/theme assets, update dry-runs, local AUR override wiring, host runtime
state, bootloader state, and recent journal errors.

Not executed in this pass:

- root-only validators requiring `sudo`
- destructive installer/storage flows
- full VM reinstall
- external CVE/vulnerability database scan
- actual package updates or AUR builds

## Release Status

Status: not release-ready yet.

The codebase is functional in many areas, but the current public baseline is
not in an optimal or releaseable state because the worktree is dirty, the
integrated shell/manifests check fails, host sync has drift, and security
warnings remain around ESP random-seed permissions and AUR builder sudo scope.

## Critical And High Findings

1. Dirty worktree blocks a reliable release baseline.

   Evidence: `git status --short` shows a large uncommitted set including theme
   artifacts, launcher scripts, Rofi/Walker configuration, generated icon theme
   migration, AUR packaging, and update scripts. `git diff --stat` reports 160
   tracked files changed with 721 insertions and 372 deletions, plus many
   untracked files.

   Impact: it is not possible to distinguish stable intentional changes from
   local experiments. Public release, VM baseline, and host sync should not be
   treated as optimal until this is reviewed, split if needed, and committed.

2. `check-shell-and-manifests` fails.

   Evidence:

   - `files/home/.local/bin/margine-launcher-rofi:119`: ShellCheck `SC2002`
   - `files/home/.local/bin/margine-launcher-rofi:191`: ShellCheck `SC2015`

   Impact: the repo's own pre-push style/static gate is red. `SC2015` is more
   than style because `A && B || C` is not equivalent to an if/else block if
   the middle command fails.

3. Host baseline drift is present.

   Evidence from `./scripts/check-host-baseline-sync`:

   - host is missing `.config/gtk-3.0/settings.ini`
   - host Rofi generated theme still uses `Margine-MoreWaita`
   - repo Rofi generated theme uses `Margine-Adwaita`

   Impact: the current host is not exactly running the repo baseline. This
   explains part of the icon-theme inconsistency seen in launchers.

4. The active Walker fix is still a local unowned binary on the host.

   Evidence:

   - `command -v walker` resolves to `/home/daniel/.local/bin/walker`
   - `/usr/bin/walker` is owned by package `walker 2.16.0-1`
   - `/home/daniel/.local/bin/walker` is not owned by any package
   - the two binaries have different SHA-256 hashes

   Impact: the scroll fix works as a host override, but this is not a clean
   distro delivery mechanism. The release path should install the patched AUR
   package, then remove or stop preferring the unowned host binary.

5. ESP random-seed permissions are a security issue.

   Evidence from `journalctl -p 3 -b`:

   - `Mount point '/boot' which backs the random seed file is world accessible`
   - `Random seed file '/boot/loader/random-seed' is world accessible`

   Impact: systemd considers this a security hole. This should be addressed in
   the boot baseline with root-level validation and an explicit remediation
   path for the ESP/random-seed permissions model.

6. `install-aur-packages` leaves a permanent NOPASSWD pacman sudo rule for the
   AUR build user.

   Evidence: `/etc/sudoers.d/90-margine-aurbuilder` generated by the script
   grants:

   - `Defaults:aurbuilder !authenticate`
   - `aurbuilder ALL=(root) NOPASSWD: /usr/bin/pacman`

   Impact: allowing a build user to run root `pacman` without authentication is
   a broad local privilege surface. It may be pragmatic for `makepkg
   --syncdeps --install`, but the rule should be temporary, tightly scoped, or
   explicitly documented as a trusted local-builder decision.

7. `hyprlock` dumped core in the current boot.

   Evidence from `journalctl -p 3 -b`: `Process ... (hyprlock) of user 1000
   dumped core` at `2026-04-23 23:00:55`.

   Impact: this is a stability finding. It is especially relevant because
   `hyprlock`, fingerprint auth, and power-key behavior are part of the laptop
   safety baseline.

8. User-facing script output is still partially Italian.

   Evidence: `update-all --dry-run`, `refresh-efi-trust`, storage/provisioning
   usage strings, and several warnings still print Italian text such as
   `Artefatti di boot`, `Verifiche finali`, `salto`, `fallito`, and
   `completato`.

   Impact: this violates the distro requirement that user-facing system output
   be English.

## Medium Findings

1. `update-all` contains an automatic `sudo pacman -Rdd --noconfirm
   geocode-glib-common` migration.

   The migration is gated by installed `geocode-glib-common` and available
   `geocode-glib`, and it addresses the observed Arch conflict. However, `-Rdd`
   bypasses dependency checks. Keep this only as a short-lived, documented
   migration, and remove it after the transition window.

2. `validate-runtime-baseline` exits with failure without reporting a precise
   failed item.

   Evidence: the verbose output showed the runtime sections, then returned
   `Result: FAIL`, but did not print the specific check that incremented
   `issue_count`.

   Impact: the validator is not actionable enough for release work.

3. Direct ShellCheck over broad globs is noisy and includes non-shell files.

   The integrated script correctly discovers Bash files by shebang, but a broad
   shellcheck run still attempts `scripts/README.md`. Add `__pycache__/` to
   `.gitignore` and keep shellcheck file discovery constrained to shebang-based
   shell files.

4. Untracked Python bytecode is present under the versioned home payload.

   Evidence: `files/home/.local/bin/__pycache__/*.pyc` is untracked.

   Impact: generated bytecode should not live in the source tree. Add ignore
   coverage and remove generated bytecode during cleanup.

5. Desktop entries are valid but have category hints.

   `desktop-file-validate` passed, but reported multi-main-category hints for:

   - `nm-connection-editor.desktop`
   - `plasma-network-management.desktop`
   - `margine-color-picker.desktop`

   Impact: minor launcher/menu duplication risk.

6. Duplicate notification providers appear in the journal.

   Evidence: DBus reports duplicate `org.freedesktop.Notifications` service
   names from SwayNC and KDE Plasma notification services.

   Impact: if KDE utilities are installed for tests, Plasma notification
   services should not compete with SwayNC in the Hyprland baseline.

7. Current host does not have the optional gaming runtime installed.

   Evidence: `gamescope`, `obs-studio`, `lact`, `vkbasalt`, and
   `obs-vkcapture` are not installed on the current public host, while the
   manifests contain them in optional gaming layers.

   Impact: not a baseline failure if the gaming layers were not selected, but a
   full gaming validation must run in a VM or host install with those layers
   enabled.

8. Package integrity checks are noisy but not immediately alarming.

   Evidence: `pacman -Qkq` reports missing runtime-managed paths for `bind`
   and `cups`, such as `/var/cache/cups/rss/` and `/var/spool/cups/tmp/`.
   `pacman -Qkkq` reports many modified configuration files, which is expected
   on a configured system.

   Impact: no immediate package breakage was proven, but a cleaner integrity
   check should classify config-file drift separately from missing package
   payload.
