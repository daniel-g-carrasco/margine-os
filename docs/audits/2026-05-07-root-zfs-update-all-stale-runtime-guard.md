# 2026-05-07 root-on-ZFS update-all stale runtime guard

## Finding

A root-on-ZFS VM could still run an older installed `update-all` runtime from
`/usr/local/lib/margine` or an older user wrapper even after the repository had
grown the root-on-ZFS fail-closed gate. That stale runtime can execute the
generic Btrfs/Snapper update path and regenerate boot artifacts that cannot
mount `/sysroot` after reboot.

## Fix

- the user `~/.local/bin/update-all` wrapper now blocks root-on-ZFS before it
  delegates to any checkout or installed runtime;
- the installed `/usr/local/bin/update-all` wrapper generated by
  `provision-user-app-config` now has the same independent root-on-ZFS guard;
- `provision-user-app-config --install-global-update-only` refreshes only the
  installed Margine runtime and wrapper;
- `repair-zfs-root-boot-chain` refreshes that installed update runtime during
  live-ISO recovery before rebuilding and validating the boot chain.

## Remaining Work

The dedicated root-on-ZFS update flow is still not implemented. Until it exists,
all `update-all` entry points must fail closed on root-on-ZFS.
